# Security Policy

## Reporting a Vulnerability

If you discover a security or privacy issue in this repository, please report it
privately rather than opening a public issue:

- **Email:** kandulanikhilvarma@gmail.com (subject line: `SECURITY`)
- Please **do not** open a public issue, pull request, or discussion for a
  security report.

Where possible, include a description of the issue, steps to reproduce, and the
affected file or component. I aim to acknowledge reports within **72 hours** and
to share a resolution timeline after an initial assessment.

## Supported Versions

This is a portfolio / research project. Only the latest state of the default
branch (`main`) is maintained — there are no backported fixes for older commits
or tags.

## Good-Faith Research

Good-faith security research is welcome. While testing, please avoid privacy
violations, service disruption, and accessing data that is not yours.
